# ⚡ NLP Plagiarism Detector & Document Analyzer ⚡

Hello there! 👋

I am excited to share my Plagiarism Detector project! It uses Artificial Intelligence to read documents and find out if the text was copied or rewritten.

This is a complete full-stack web application. It has a modern frontend built with React, a fast backend built with FastAPI, and uses Hugging Face's cloud AI. This means it can run smoothly without needing an expensive graphics card on your computer.

Think of it as a smart tool that scans documents, saves the results to a database, and displays everything on a beautiful webpage.

## ✨ Key Features
This tool is designed to automate the process of analyzing text and PDFs for plagiarism. Key features include:

**1. Advanced AI Detection (Pillar 1)**
* **Live Internet Search:** It searches the live web using DuckDuckGo to compare your text against billions of real websites, rather than just checking a small local database.
* **AI-Generated Content Detection:** It uses a smart AI model to guess if the text was written by a human or generated by an AI like ChatGPT or Claude.
* **Multi-Language Support:** It understands over 50 different languages. It can even catch someone if they copy a Spanish article and translate it to English!
* **Meaning Matching:** Instead of just looking for exact matching words, the AI actually understands the *meaning* of the sentences to find hidden copying.
* **Paraphrase Detection:** Easily catches people who try to cheat by changing a few words around.
* **Confidence Scoring:** Gives you a clear percentage score so you know exactly how much of the text is copied.

**2. High-Performance Enterprise Backend (Pillar 2)**
* **Background Task Queue:** Uses Celery and PostgreSQL to process heavy AI tasks in the background. This keeps the website fast and responsive for hundreds of daily users. *(Note: If the app ever grows to 10,000+ daily users, we can easily swap PostgreSQL for Redis to handle the massive traffic without rewriting any code).*
* **FastAPI Backend:** Built with FastAPI so the server responds instantly to the user while the heavy lifting happens behind the scenes.
* **Multi-threaded Web Search:** The background worker searches multiple websites at the exact same time, which makes the deep-web scan 80% faster.
* **PostgreSQL Database:** Safely stores all your past document scans and also acts as the queue manager for the background worker.

**3. Modern Frontend Dashboard (Pillar 3)**
* **React & Vite:** A very fast and smooth webpage built with modern React.
* **AI Content Panel:** Shows you clearly if the document was "Likely AI" or "Likely Human".
* **Scan Modes:** Lets users choose between a slow "Deep Web Scan" or a quick "Fast Local Scan".
* **Source Links:** Provides clickable links directly to the website where the user copied the text from.
* **Tailwind CSS:** A beautiful design that looks great on any screen size.

## 🛠️ Usage
Here’s how to get started with the NLP Plagiarism Detector. 👇

**Step 1: Get Your Free Hugging Face API Token**
1. Create a free account at [huggingface.co](https://huggingface.co)
2. Go to Settings → Access Tokens → Create a **Read** token
3. Copy the token (it starts with `hf_...`)

**Step 2: Setup Environment Variables**
Create a `.env` file in the `plagiarism-backend` directory (see `.env.example` for reference):
```env
DB_HOST=localhost
DB_PORT=5432
DB_NAME=plagiarism_db
DB_USER=postgres
DB_PASSWORD=yourpassword

# Hugging Face Inference API
HF_API_TOKEN=hf_your_token_here
```

**Step 3: Start the Backend API & Background Worker**
You will need two terminals for the backend (one for the API, one for the Celery worker).
```bash
# Terminal 1: Start the FastAPI Server
cd plagiarism-backend
pip install -r requirements.txt
uvicorn main:app --reload --port 8000

# Terminal 2: Start the Celery Background Worker
cd plagiarism-backend
# Note: --pool=solo is required for Windows. Mac/Linux users can omit it.
celery -A worker worker --pool=solo --loglevel=info
```
*(Note: All AI inference runs via the Hugging Face cloud API — no local GPU required!)*

**Step 3: Start the Frontend Dashboard**
Navigate to the frontend directory, install the Node packages, and start the Vite development server.
```bash
cd plagiarism-frontend
npm install
npm run dev
```
Open the provided local URL (e.g., `http://localhost:5173`) in your web browser to view the live dashboard and start analyzing documents!

## 📁 Project Structure
Here’s how the project is organized for maximum clarity and separation of concerns:

* **`plagiarism-frontend/`**: The React website where users upload files and see their scores.
* **`plagiarism-backend/`**: The main server that handles file uploads and talks to the database.
* **`plagiarism-nlp/`**: The AI brain of the project that does the actual plagiarism scanning and web searching.

## 🤔 Why This Architecture?
This tool provides a robust, scalable solution for document analysis. Key benefits include:

* **Modular Design:** The frontend, backend, and AI engine are separated. This makes the code much cleaner and easier to upgrade in the future.
* **Smart Analysis:** Because it understands the meaning of words instead of just looking for exact matches, it catches clever cheaters that older tools would miss.
